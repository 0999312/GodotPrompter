# Full-Depth Self-Verification Protocol

GodotPrompter Self-Verification is the mechanism by which agents validate their own generated code by running GUT tests against Godot's `--headless` mode. This leverages the agent's unlimited stamina — it can loop: generate → test → fail → fix → retest until all criteria pass.

---

## Verification Hierarchy

```
Layer 1: Automated Checks (agent runs, no user needed)
  └─ Code static analysis (grep patterns, file format checks)
  └─ GUT test execution (godot --headless + GUT CLI)
  └─ Type safety scan (untyped declarations, Variant traps)

Layer 2: Manual Checks (agent reviews code, reports to user)
  └─ Architecture compliance (signal direction, node hierarchy)
  └─ Race condition inspection (_unhandled_input, deferred calls)
  └─ Comment/dead code audit

Layer 3: Behavioral Checks (user must test in actual Godot)
  └─ GUT test suite (run by agent, read by user)
  └─ Play test scenario (described by skill, executed by user)
```

## GUT Integration

### Prerequisites

The target project must have GUT installed in `addons/gut/`. If not:

1. Download GUT from [gut-community/gut](https://github.com/gut-community/gut)
2. Extract to `addons/gut/`
3. Enable in Project Settings → Plugins → GUT → Enable

### Test Directory Structure

```
res://tests/
  skill_verification/          # Agent-generated tests (created on demand)
    player_controller_move.gd
    state_machine_transition.gd
    ... 
  project/                     # Existing project tests (if any)
```

### Test File Naming

```
<skill-name>_<criterion-id>.gd
Example: player_controller_move_speed.gd
```

### Test Script Template

```gdscript

# Generated by verification protocol for skill: player-controller
# Criterion: Movement precision — player reaches exactly speed px/s

const PROJECT_PATH := "res://"
const PLAYER_SCENE := preload("res://scenes/player/player.tscn")

var player: CharacterBody2D


func before_each() -> void:
	player = PLAYER_SCENE.instantiate()
	add_child_autofree(player)
	# Set up the test scene
	player.global_position = Vector2(100, 100)


func test_movement_speed() -> void:
	var speed: float = player.speed  # read from player's exported var
	var direction := Vector2.RIGHT
	
	player._process(speed)  # or _physics_process with speed-adjusted delta
	
	# After 1 second of moving at speed px/s, position should have changed by speed
	var expected_delta := speed * 1.0
	var actual_delta := player.global_position.x - 100.0
	
	assert_almost_eq(actual_delta, expected_delta, 1.0,
		"Player should move by speed units in 1 second")
```

### Running Tests

```bash
# Run all skill verification tests
godot --headless --path <project_directory> \
  -s addons/gut/gut_cmdln.gd \
  -gdir=res://tests/skill_verification \
  -gprefix=skill_ \
  -gexit

# Run a single test file
godot --headless --path <project_directory> \
  -s addons/gut/gut_cmdln.gd \
  -gselect=res://tests/skill_verification/player_controller_move.gd \
  -gexit

# Run tests with XML output for parsing
godot --headless --path <project_directory> \
  -s addons/gut/gut_cmdln.gd \
  -gdir=res://tests/skill_verification \
  -gxml=res://tests/results.xml \
  -gexit
```

### Parsing Results

GUT exit codes:
- `0`: All tests passed
- `1`: Some tests failed

When exit code is `1`, the agent must:
1. Read the console output (or XML file)
2. Identify the failing test(s) by name
3. Read the test assertion that failed
4. Diagnose the root cause in the generated code
5. Fix the code
6. Re-run

## Verification Loop Protocol

```
START
  │
  ▼
Generate implementation code
  │
  ▼
Generate GUT test script(s) from Success Criteria
  │
  ▼
Run Godot headless with GUT
  │
  ├── Exit code 0 (ALL PASS) ───► Report completion
  │
  └── Exit code 1 (SOME FAIL)
        │
        ▼
      Loop counter += 1
        │
        ├── Counter ≤ 5 ───► Read failure → diagnose → fix → RE-RUN
        │
        └── Counter > 5 ───► Report: "Self-verification did not pass
                              after 5 attempts. Remaining failures:
                              [list]. Requesting human guidance."
                               END
```

### Loop Limit & Escalation

| Attempt | Action |
|---------|--------|
| 1 | Fix and re-run silently |
| 2 | Fix and re-run silently |
| 3 | Output warning: "Test still failing at attempt 3. Trying [approach]." |
| 4 | Ask user: "Tests failing after 4 attempts. May I try a different approach? [suggest 1-2 alternatives]" |
| 5 | Escalate: "Could not pass Self-Verification after 5 attempts. Output below. Please advise." |

### Verification During Implementation

For each code generation step:
1. **Before writing code**: Read the skill's Success Criteria
2. **After writing code**: Run verification loop
3. **After passing verification**: Proceed to next step

## Type Safety Scan (Post-Generation)

In addition to GUT tests, the agent must scan all generated GDScript for:

```bash
# Check 1: Bare := with Variant-returning APIs
rg ':=.*\.(get|call)\(|:=.*JSON\.|:=.*str_to_var|:=.*bytes_to_var|:=.*load\(|:=.*get_node\('

# Check 2: Untyped function parameters
rg 'func \w+\([^)]*:[^)]'  # should have : Type on every param

# Check 3: Untyped @export
rg '@export var \w+(?!\s*:)' 

# Check 4: Untyped for-loop on collection
rg 'for \w+ in \w+:(?!.*Array)'  # should have typed collection
```

## Scene File Validation

```bash
# Check: ext_resource declared BEFORE sub_resource in .tscn files
rg -n '^\[ext_resource' *.tscn | head -1
rg -n '^\[sub_resource' *.tscn | head -1
# First ext_resource line number MUST be < first sub_resource line number
```

## Protocol Constraints

1. **Do NOT skip verification** because "it's a small change" — every change runs verification
2. **Do NOT suppress test failures** — fix the root cause, not the test
3. **Do NOT modify the test** to match the code — modify the code to match the test
4. **Do NOT run more than 5 loops** — escalate to human
5. **Do NOT delete tests after passing** — leave them in `tests/skill_verification/` for future regression
