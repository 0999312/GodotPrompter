# Variant Inference Strict Typing Hardening / Variant 推断严格类型化加固

## English (Primary)

### Problem
Generated GDScript frequently triggers the Godot 4.4+ / 4.6.2 warning:
`The variable type is being inferred from a Variant value, so it will be typed as Variant. (Warning treated as error.)`
With "Warnings as Errors" enabled (default in many CI setups, recommended for Godot 4.6.2 projects), this fails the build.

### Root Causes
The `:=` walrus-style inference operator is overused. When the right-hand side returns `Variant`, the variable becomes `Variant` and triggers the warning. Variant-returning APIs include:
- `Dictionary.get(key)` and untyped `Dictionary[key]`
- `JSON.parse_string`, `str_to_var`, `bytes_to_var`
- `Object.get`, `Object.call`, `callv`, `get_meta`
- `load(...)` (returns broad `Resource`), `ResourceLoader.load(...)`
- `get_node(...)` / `get_node_or_null(...)` without class hint
- `get_children()` of untyped containers
- Untyped signal handler / lambda parameters
- `@export var x` without type annotation
- `for x in untyped_array`
- `match` value bindings without annotation

### Godot 4.6.2 Context
- Stricter static type enforcement compared to 4.3
- Typed Dictionaries (`Dictionary[K, V]`) are stable since 4.4
- Project setting `Debug → GDScript → Warnings` exposes per-warning severity; `untyped_declaration` and `inferred_declaration` should be set to `error` for new projects
- `@warning_ignore("inference")` should be a last resort, not a workaround

## 中文（镜像）

### 问题
生成的 GDScript 经常在 Godot 4.4+ / 4.6.2 中触发警告：
`The variable type is being inferred from a Variant value, so it will be typed as Variant. (Warning treated as error.)`
在启用 "Warnings as Errors"（许多 CI 默认开启，Godot 4.6.2 项目推荐配置）的情况下，该警告会直接导致构建失败。

### 根因
`:=` 推断运算符被过度使用。当右值返回 `Variant` 时，变量也被推断为 `Variant`，触发警告。返回 `Variant` 的常见 API 包括：
- `Dictionary.get(key)` 与未类型化的 `Dictionary[key]`
- `JSON.parse_string`、`str_to_var`、`bytes_to_var`
- `Object.get`、`Object.call`、`callv`、`get_meta`
- `load(...)`（返回宽泛的 `Resource`）、`ResourceLoader.load(...)`
- 未标注类的 `get_node(...)` / `get_node_or_null(...)`
- 未类型化容器上的 `get_children()`
- 未标注类型的信号处理器 / lambda 参数
- 未标注类型的 `@export var x`
- `for x in untyped_array`
- 未标注类型的 `match` 值绑定

### Godot 4.6.2 背景
- 相对 4.3 收紧了静态类型检查
- 类型化字典（`Dictionary[K, V]`）自 4.4 起稳定可用
- 项目设置 `Debug → GDScript → Warnings` 中暴露各警告等级，新项目建议把 `untyped_declaration` 和 `inferred_declaration` 设为 `error`
- `@warning_ignore("inference")` 应作为最后手段而非通用绕过方式
